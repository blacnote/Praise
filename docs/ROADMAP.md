# ChurchSense — Roadmap

> Core rule: ChurchSense follows worship. Worship does not follow ChurchSense.
> Source of truth: `ChurchSense_Phase1_Blueprint.md`.

## Phase 1 (current focus) — the trustworthy Sunday foundation
- Worship Set Builder (add songs, order, total/elapsed/remaining time, jump to song)
- Song data (title, artist, key, BPM, lyrics, sections + lengths)
- Stem playback (drums, bass, guitar, keys, pads, FX, BGV, percussion — **no lead vocal**)
- Click + guide (count-ins, section cues) — **team ears only, never the house**
- Audio routing concept (house / click / guide / stem groups / stereo bounce → StudioLive 64)
- Route into the church's **existing** monitors/in-ears (no custom hardware)
- ProPresenter sync (lyrics/slides follow the live flow)
- **Spirit Flow Mode** — hold, loop, prayer/pad, resume; live time recalculation

## Not in Phase 1
Custom in-ear hardware · per-member monitor mixing · building our own stem separation ·
lighting/video control · recording · multi-site sync · mobile team remote · song marketplace.

## Build-order (the honest bet)
1. **Prove Spirit Flow audio core** — glitch-free hold/loop of click+guide+stems. Riskiest, most differentiating.
2. Wrap it in Set Builder + Live view (v1 — done as POC).
3. Real multi-stem audio file playback (replace synth tones).
4. Audio routing layer (desktop) for true separate outputs.
5. Real ProPresenter network sync (resilient, degrades gracefully).

## Future phases
- **P2** Personal monitor mixing (each team member's own in-ear blend)
- **P3** Deeper ProPresenter / worship moments / Scripture cues
- **P4** Custom in-ear / hardware exploration (only if needed)
- **P5** Stem preparation/import tools
- **P6** Service recording & review
- **P7** Multi-site / shared library + roles
- **P8** Team mobile companion
